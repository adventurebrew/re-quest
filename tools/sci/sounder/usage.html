Quick Start

You should select an input file to work on (multiple files can be selected, either with *, or with the dialog).

Usually, you'll want to leave the input version as AUTO_DETECT, unless there are problems.

Now, you can choose to do various things with the file (you can select any combination, by default nothing is done):

  • Play file's musical contents (ignoring digital sample)
  • Save file's contents in various formats
  • Add digital sample (or replace it, if it already exists)
  • Play digital sample
  • Save only the digital sample

When you've selected the required actions, click Start.

Now, you can either Restart to repeat the exact same actions, Edit to go back and modify some settings, or Close to finish.

Input

The input file can be one of:
  • Early SCI0 (rare)
  • SCI0
  • SCI1+
  • Standard MIDI File
It's recommended to use AUTO_DETECT. If something is weird, try and set it explicitly, it might improve the error message.

Play

After a file has been selected, and play has been checked, it's possible to select the play device - each device (Adlib/MT-32/etc.) has channels associated with it, this selects which channels to play (or all channels in file). It doesn't try to emulate that sound device, it merely selects the sound device's relevant channels.
Note: if there's a problem in reading the file, 'play device' will show every device known to Sounder.

By default the music will be played through Microsoft GS Wavetable Synth (on Windows). Sadly, this device isn't perfect. You might want to install FluidSynth, or similar program, to add more ports. Furthermore, it's recommended to install munt in order to have an MT-32 emulation, if you want to hear its channels.

The port can be changed to any MIDI port recognized by your OS.

Save

The file can be saved in any of the following formats, regardless of its original format:
  • Early SCI0 (rare)
  • SCI0
  • SCI1+
  • Standard MIDI File

If not save format is chosen, no file will be saved.

After selecting save format, the other save-related options can be chosen.

If the input file had a digital sample, or a digital sample was loaded (in input wav), and the file isn't saved as Standard MIDI File - then the digital sample will be saved as part of the file.

You can supply a name (and path) for the save file, or leave that field empty, and use the default - which is adding .snd for any Sierra format, and .mid for MIDI files - even if that extension already existed. e.g., if the original file is 100.snd it will be saved as 100.snd.snd

If such file exists, it will be overwritten without further notice!

Similarly to the play device option, you can control the save midi device. Those settings aren't related to each other.

Digital

Digital sample can come from two places:
  • Existing in input file
  • Loaded from input digital file
If input digital file is supplied, it will replace the digital sample if it existed in the input file.

The input digital file can be of any type. If it's not an unsigned 8 bit uncompressed .wav file, it will be converted to such format. The frequency will be 22000Hz if it was so, or higher, and 11000Hz otherwise.

Note: in ScummVM it played fine, but if using 22KHz in older games is a problem in original Sierra interpreter, just report to me, and I will change it.

The digital sample can be played (or saved), regardless of the music from the input file. Similar rules apply to save name and overwriting.

Misc

In GUI mode, by default, file's information is printed. In CLI it's off by default.

Command Line Interface

The Graphical User Interface (GUI) and Command Line Interface (CLI) for Sounder have same capabilities. But sometimes using the one is more convenient than using the other.

The Windows release contains two executables: sounder.exe and sounder_no_console.exe. They are almost identical, except that the sounder_no_console.exe is optimized for GUI, and doesn't send anything to the console, even if started from one. On the other hand, GUI can be started from sounder.exe, but it will also open the ugly command prompt window.

To summarize, it's recommended to use sounder_no_console.exe for GUI (and it's used by the Sounder shortcut), and sounder.exe for CLI.

All this discussion isn't relevant if you're using Linux, or running sounder.py directly.

If sounder.py/sounder.exe/sounder_no_console.exe is started without any parameters, the GUI will be started. If it's called with any parameter, CLI will be started. To learn how to use CLI, just call it with --help. The sounder command line shortcut refers to sounder.bat which does exactly that.
